# Asuri Major 竞技场规则手册

## 一、赛制概述

本题为 **2v2 强化学习对抗**。两支队伍分别控制：

- **进攻方 T（attack）**
- **防守方 CT（defense）**

T 的目标是进入包点完成下包并等待爆炸；CT 的目标是阻止下包，或在下包后完成拆弹。

每支队伍提交一个 `.safetensors` 权重文件。权重加载后固定，比赛时不允许运行时通信。

---

## 二、选手下载物

平台提供以下文件下载：

- `KOH_rules.md`：本规则文件
- `koh_env.py`：本地训练与调试环境
- `koh_baseline_template.py`：可直接运行的 baseline

建议本地准备方式：

```bash
python koh_baseline_template.py --role attack --maps-dir maps --episodes 3000 --output attack.safetensors
python koh_baseline_template.py --role defense --maps-dir maps --episodes 3000 --output defense.safetensors
```

如果你只想在单张地图上训练，也可以：

```bash
python koh_baseline_template.py --role attack --map maps/Ascent亚海悬城.txt
```

## 三、地图与地图文件格式

### 3.1 地图规格

- 地图大小固定为 **25 x 25**
- 坐标表示为 `(row, col)`
- 左上角为 `(0, 0)`
- `row` 向下增大，`col` 向右增大

### 3.2 ASCII 地图文件

每个下载到的 `.txt` 地图文件都是一张 ASCII 地图，每行 25 个字符，共 25 行。

字符含义如下：

| 字符 | 含义 |
|:---:|:---|
| `.` | 空地 |
| `#` | 障碍物 |
| `A` | A 包点 |
| `B` | B 包点 |
| `T` | T 出生点 |
| `C` | CT 出生点 |

合法地图必须满足：

- 恰好 **2 个 `T`**
- 恰好 **2 个 `C`**
- 恰好 **1 个 `A`**
- 恰好 **1 个 `B`**

### 3.3 地图机制

障碍物同时具备三种作用：

- 阻挡移动
- 阻挡视线
- 阻挡弹道

赛事轮次会从管理员维护的地图池中生成对局地图。选手本地训练时，可在地图详情页按需下载单张地图文件进行训练。

## 四、回合结构

- 每局最多 **200 步**
- 若炸弹已经安放，则不再受 200 步限制，最多继续等待炸弹倒计时结束或被拆除
- 每一步双方同时为己方两名角色提交动作
- 动作**同步结算**，无先手优势
- **步内结算顺序：射击判定先于移动判定执行**
  - 射击命中基于本步开始时（移动前）的位置快照
  - 即使目标角色本步选择了移动动作，也无法通过移动来闪避已判定的射击
  - 移动在射击结算（含死亡更新）完成后执行

---

## 五、动作空间

每名角色每步执行 1 个动作，共 9 种：

| 索引 | 名称 | 效果 |
|:---:|:---|:---|
| 0 | 上移 | 向 `row-1` 移动 1 格 |
| 1 | 下移 | 向 `row+1` 移动 1 格 |
| 2 | 左移 | 向 `col-1` 移动 1 格 |
| 3 | 右移 | 向 `col+1` 移动 1 格 |
| 4 | 停留 | 原地不动，同时可触发下包/拆弹进度 |
| 5 | 向上开火 | 向上直线射击 |
| 6 | 向下开火 | 向下直线射击 |
| 7 | 向左开火 | 向左直线射击 |
| 8 | 向右开火 | 向右直线射击 |

---

## 六、核心机制

### 6.1 移动

- 每个格子同一时刻最多容纳 1 名存活角色
- 目标格越界、为障碍物、或本步结算后发生冲突时，移动失败并原地停留
- 多名角色同时抢同一目标格时，相关移动全部失败

### 6.2 开火与击杀

- 开火为 **hit-scan** 判定
- 射程最远 **5 格**
- 只支持上下左右四个正交方向
- 子弹遇到障碍物或边界立即停止
- 命中即死亡，无生命值系统
- 开火后进入 **2 步冷却**
- 同一步允许互相击杀（trade）

说明：

- 视线检测不限制为上下左右或 45° 对角线
- 但开火仍然只支持 4 个正交方向
- **射击判定先于移动执行**：命中判定基于本步开始时所有角色的位置，移动动作对本步射击无效。即：目标选择移动也无法躲避本步已判定的子弹，反之，射击方移动后也无法影响本步射击的起始位置。

### 6.3 下包

下包方为 T：

- T 站在 `A` 或 `B` 包点上
- 当步动作必须为 `stay`（索引 4）
- 连续停留 **3 步**完成下包
- 中断、移动、开火或离开包点会重置当前包点进度
- A/B 进度独立计算
- 任一包点完成后，炸弹进入 **30 步倒计时**

### 6.4 拆弹

拆弹方为 CT：

- 炸弹已安放
- CT 站在炸弹所在格
- 当步动作必须为 `stay`（索引 4）
- 连续停留 **3 步**完成拆弹
- 中断后进度归零

---

## 七、胜负判定

每步结算后，按以下优先级判定：

| 优先级 | 条件 | 结果 |
|:---:|:---|:---|
| 1 | 炸弹倒计时归零 | T 胜 |
| 2 | CT 完成拆弹 | CT 胜 |
| 3 | 炸弹未安放时 T 全灭 | CT 胜 |
| 4 | CT 全灭 | T 胜 |
| 5 | 达到 200 步且炸弹未安放 | CT 胜 |

边界说明：

- 下包后 T 全灭，对局不会立刻结束，仍继续等待爆炸或拆弹
- 下包后双方同归于尽，按优先级仍判定 **T 胜**

## 八、观测空间

每支队伍每步获得一个 **77 维**浮点向量：

- 两个角色块，各 **36 维**
- 全局信息 **5 维**

所有绝对坐标按 `24.0` 归一化到 `[0, 1]`，相对位移归一化到 `[-1, 1]`。

归一化方式如下：

- 绝对坐标：`row_norm = row / 24.0`，`col_norm = col / 24.0`
- 相对位移：`delta_row = (target_row - self_row) / 24.0`，`delta_col = (target_col - self_col) / 24.0`
- 例：自己在 `(10, 8)`，目标在 `(13, 2)`，则相对位移为 `(3/24, -6/24)`

### 8.1 单个角色的 36 维块

| 偏移 | 字段 | 说明 |
|:---:|:---|:---|
| 0 | `alive` | 自己是否存活。存活为 `1.0`，死亡为 `0.0` |
| 1 | `self_row` | 自己所在行，取值为 `self_row / 24.0` |
| 2 | `self_col` | 自己所在列，取值为 `self_col / 24.0` |
| 3 | `site_A_delta_row` | A 包点相对自己的行偏移，取值为 `(A_row - self_row) / 24.0` |
| 4 | `site_A_delta_col` | A 包点相对自己的列偏移，取值为 `(A_col - self_col) / 24.0` |
| 5 | `site_B_delta_row` | B 包点相对自己的行偏移，取值为 `(B_row - self_row) / 24.0` |
| 6 | `site_B_delta_col` | B 包点相对自己的列偏移，取值为 `(B_col - self_col) / 24.0` |
| 7 | `teammate_delta_row` | 队友相对自己的行偏移；队友死亡时为 `0.0` |
| 8 | `teammate_delta_col` | 队友相对自己的列偏移；队友死亡时为 `0.0` |
| 9 | `teammate_alive` | 队友是否存活。存活为 `1.0`，死亡为 `0.0` |
| 10 | `enemy1_delta_row` | 敌人 1 相对自己的行偏移；敌人 1 不可见或死亡时为 `0.0` |
| 11 | `enemy1_delta_col` | 敌人 1 相对自己的列偏移；敌人 1 不可见或死亡时为 `0.0` |
| 12 | `enemy1_visible` | 敌人 1 是否可见。可见为 `1.0`，不可见或死亡为 `0.0` |
| 13 | `enemy2_delta_row` | 敌人 2 相对自己的行偏移；敌人 2 不可见或死亡时为 `0.0` |
| 14 | `enemy2_delta_col` | 敌人 2 相对自己的列偏移；敌人 2 不可见或死亡时为 `0.0` |
| 15 | `enemy2_visible` | 敌人 2 是否可见。可见为 `1.0`，不可见或死亡为 `0.0` |
| 16-23 | `ray_*` | 从自己位置向 8 个方向探测墙或边界，见下方说明 |
| 24 | `on_cooldown` | 自己是否处于开火冷却。冷却中为 `1.0`，否则为 `0.0` |
| 25-33 | `action_mask_*` | 动作可用性提示，顺序对应动作 `0-8`，见下方说明 |
| 34 | `path_dist_to_A` | 自己到 A 包点的最短路距离比例，见下方说明 |
| 35 | `path_dist_to_B` | 自己到 B 包点的最短路距离比例，见下方说明 |

`ray_*` 的方向顺序为：

1. 上
2. 下
3. 左
4. 右
5. 左上
6. 左下
7. 右上
8. 右下

每个 `ray_*` 表示对应方向上离墙或地图边界有多近。计算方式为：从自己所在格出发，沿该方向数连续空格数量 `d`，直到遇到障碍物或地图边界；该字段取值为 `1 / (d + 1)`。

例：如果自己正上方第一格就是墙，则 `d = 0`，向上 ray 为 `1.0`；如果向上可以走 3 格，第 4 格才是墙或边界，则向上 ray 为 `1 / 4 = 0.25`。

`action_mask_*` 的顺序与动作编号一致：

1. 上移
2. 下移
3. 左移
4. 右移
5. 停留
6. 向上开火
7. 向下开火
8. 向左开火
9. 向右开火

每个 mask 字段为 `1.0` 或 `0.0`：

- 移动动作：目标格在地图内、不是障碍物、且当前没有存活角色占据时为 `1.0`，否则为 `0.0`
- 停留动作：存活时为 `1.0`，死亡时为 `0.0`
- 开火动作：角色存活且不在开火冷却中时为 `1.0`，否则为 `0.0`

注意：`action_mask_*` 只是帮助模型减少无效探索，不代表该动作一定是最优动作。例如开火 mask 为 `1.0` 只表示当前可以开火，不表示该方向一定能命中敌人。

`path_dist_to_A` 与 `path_dist_to_B` 表示从自己当前位置走到对应包点的最短路长度，障碍物不可穿过。取值为 `min(distance, 48) / 48.0`；如果角色死亡或包点不可达，则为 `1.0`。

敌人可见条件：

1. 曼哈顿距离不超过 `10`
2. 自己所在格中心点与敌人所在格中心点之间的连线没有被障碍物遮挡

遮挡判定：如果这条连线与任意障碍格相交，则视为被遮挡，敌人不可见。可见性不要求双方处在同一行、同一列或 45° 对角线上。

### 8.2 全局 5 维

| 索引 | 字段 | 说明 |
|:---:|:---|:---|
| 72 | `site_A_progress` | A 包点下包进度比例；若炸弹已下在 A 包点，则保持 `1.0`，否则为当前 A 包点下包进度 |
| 73 | `site_B_progress` | B 包点下包进度比例；若炸弹已下在 B 包点，则保持 `1.0`，否则为当前 B 包点下包进度 |
| 74 | `defuse_progress` | 当前拆弹进度比例，`0.0` 表示没有进度，`1.0` 表示已完成拆弹 |
| 75 | `bomb_timer_ratio` | 炸弹剩余时间比例；未下包时为 `0.0`，刚完成下包时为 `1.0`，爆炸前逐步降到 `0.0` |
| 76 | `remaining_steps` | 常规 200 步的剩余比例；第 0 步为 `1.0`，到达 200 步时为 `0.0` |

说明：`site_A_progress` 与 `site_B_progress` 共同表达下包状态。未下包时，它们分别表示 A/B 两个包点当前的下包进度；下包完成后，炸弹所在包点对应字段为 `1.0`，另一个包点为 `0.0`。例如 `[1.0, 0.0]` 表示炸弹已下在 A 包点，`[0.0, 1.0]` 表示炸弹已下在 B 包点。模型输入维度仍保持 **77 维** 不变。

## 九、模型输出协议

模型输出固定为 **18 个 logit**：

- `output[0:9]` 控制角色 1
- `output[9:18]` 控制角色 2

每段各自取 `argmax`，得到动作索引 `0-8`。

死亡角色的动作会被环境忽略。

---

## 十、提交规范

提交模型必须匹配以下网络结构：

```python
import torch.nn as nn

class DQN(nn.Module):
    def __init__(self, input_dim=77, output_dim=18):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, 256), nn.ReLU(),
            nn.Linear(256, 256), nn.ReLU(),
            nn.Linear(256, output_dim),
        )

    def forward(self, x):
        return self.net(x)
```

要求：

- 文件格式：`.safetensors`
- 文件大小：不超过 `50 MB`
- `attack` 与 `defense` 分别提交
- 系统会按固定结构加载参数，结构不匹配会被拒绝

## 十一、Baseline 使用说明

平台提供的 `koh_baseline_template.py` 已支持地图载入：

```bash
python koh_baseline_template.py --role attack --maps-dir maps --episodes 3000
python koh_baseline_template.py --role defense --maps-dir maps --episodes 3000
```

支持参数：

- `--map <file>`：只使用单张地图训练
- `--maps-dir <dir>`：读取目录下全部 `*.txt` 地图并在每局随机采样
- `--seed <int>`：设置随机种子
- `--output <file>`：指定输出权重路径

必须传入 `--map` 或 `--maps-dir`，baseline 只会从下载的地图文件中采样训练地图。

建议优先使用比赛中实际会用到的地图文件进行训练。
